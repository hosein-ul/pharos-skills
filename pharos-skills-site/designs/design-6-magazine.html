SKILL · 01

pharos-tx-guardrail

A pre-execution checkpoint, in six movements.

Every write transaction on Pharos passes through this skill first. It simulates the call, checks the target contract, decodes the selector, scans for dangerous calldata patterns, weighs the value against the wallet balance, and estimates the gas. The output is a single number — a risk score between zero and one hundred — and a clear instruction: PROCEED, WARN, or BLOCK.

The rules favour caution. Unlimited approvals are hard-blocked at seventy points or more. Missing contract code raises the floor to eighty-five. Ownership transfers and upgrade calls earn a thirty-five point penalty before any other check runs.

→ simulation .......... PASS → contract_exists ..... PASS USDC, 1798 bytes → selector ............ approve(address,uint256) → unlimited_approval .. FAIL MAX_UINT256 → value_check ......... PASS → gas_estimate ........ PASS 71,897 gas → RISK 70/100 — HIGH — BLOCK

SKILL · 02

pharos-rwa-yield-router

A read-only yield scanner, in nine voices.

Where the guardrail decides whether a transaction is safe, the yield router decides whether it is worthwhile. It reads live APY from the entire Pharos RWA stack — R25 Axil, Zona, Morpho Blue, TermMax, AquaFlux, Faroo — and multiplies each by an asset-class risk factor between zero-point-five and zero-point-nine-five.

The honest ranking includes protocols deployed but not yet active. The router never invents APY for a vault with no markets; it says so, plainly, and continues.

→ scanning 9 protocols on mainnet 1672 ... → #1 R25 Axil 7-day 8.5% × 0.60 = 5.10% → #2 R25 Axil 6-month 11.2% × 0.60 = 6.72% → #3 Zona USDC supply 3.6% × 0.80 = 2.88% → ⏳ Morpho · TermMax · AquaFlux · Faroo → best @ $5K → R25 7d → ~$34.93 / 30d

II
— The Directory —

Twelve agents, each in its own voice.

Pages 08 — 23
$
auto-detects every installed agent — or add: | bash -s -- --for hermes

The skills are written once, in the Pharos Skill Engine format. Where they are read from depends on the agent. Three runtimes — the Pharos Skill Engine itself, Anthropic's Claude Code, and Anvita Flow — host SKILL.md files natively and trigger them on intent. The remaining nine — Cursor, Windsurf, Cline, Continue.dev, Aider, Codex CLI, Gemini CLI, GitHub Copilot, and Hermes/Ollama local models — accept SKILL.md as workspace context, custom commands, or system prompt.

Each entry in the directory below names the runtime, prints its official mark, and lists exactly one command. No build steps. Foundry's cast is the only binary either skill assumes on your PATH.

— Onboarding —

From zero to first scan, in four chapters.

I.

Install Foundry

Both skills shell out to cast for read-only RPC calls.

curl -L https://foundry.paradigm.xyz | bash foundryup cast --version
II.

Clone the bundle

One repo, both skills. Pick the path that matches your agent runtime.

git clone \ https://github.com/hosein-ul/pharos-skills \ ~/pharos-skills
III.

Verify a live read

Confirm RPC, oracle, and your local cast install all work.

cast call 0x9356C87a... \ "latestAnswer()(int256)" \ --rpc-url https://rpc.pharos.xyz
IV.

Guarded transaction

Prompt your agent. Watch the guardrail fire before signing.
