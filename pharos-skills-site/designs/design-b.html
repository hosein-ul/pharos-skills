The bundle

Two skills that compose into a workflow.

The yield router decides where idle capital should go. The tx guardrail verifies the call is safe before it's signed. Together, they cover the full lifecycle of a Pharos transaction.

SKILL · 01

pharos-tx-guardrail

Pre-execution safety checkpoint. Runs before every cast send. Returns a risk score 0–100 with PROCEED / WARN / BLOCK.

  • 6 read-only checks — simulation, contract existence, selector decode, calldata pattern, balance ratio, gas estimate
  • Hard-block on unlimited approvals (MAX_UINT256) and missing contract code
  • Dangerous-selector DB covers ownership transfer, upgrade, set-admin patterns
  • ~120 ms per call · zero on-chain writes · zero gas

SKILL · 02

pharos-rwa-yield-router

Read-only yield scanner across Pharos's RWA stack. Live APY, risk-adjusted ranking, capital-tier recommendation.

  • 9 protocols tracked — R25 Axil, Zona, Morpho Blue, TermMax, AquaFlux, Faroo + 3 more
  • Asset-class risk multipliers (0.50 → 0.95) per protocol category
  • Filters by user tolerance, min deposit, lockup window, capital size
  • Honest status — flags protocols deployed but not yet active

Onboarding

From zero to first scan in four minutes.

No build step. No compiler. The skill is markdown + JSON that your agent reads at runtime. You only need Foundry's cast on your PATH.

1

Install Foundry

Both skills shell out to cast for read-only RPC.

curl -L https://foundry.paradigm.xyz | bash foundryup cast --version
2

Clone the bundle

One repo, both skills. Path depends on your agent.

git clone \ https://github.com/hosein-ul/pharos-skills \ ~/pharos-skills
3

Verify a live read

Confirm RPC + oracle + cast all work end-to-end.

cast call 0x9356C87a... \ "latestAnswer()(int256)" \ --rpc-url https://rpc.pharos.xyz
4

Run a guarded tx

Ask your agent something Pharos-related. Watch the guardrail fire.
