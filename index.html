Security Skill

How pharos-tx-guardrail works

Six sequential read-only checks run in under 3 seconds. Score accumulates; override rules enforce minimums for critical patterns.

CHECK 01
Simulation
eth_call with --from sender. Detects transactions that would revert before they hit the chain.
+20 if reverts
CHECK 02
Contract Check
eth_getCode — empty = sending to nothing. Under 100 bytes = proxy stub or honeypot.
+50 / +15
CHECK 03
Selector Decode
cast 4byte lookup + known table. Ownership transfers, proxy upgrades, forced burns → flagged.
+35 dangerous
CHECK 04
Unlimited Approval
Pattern match: approve(addr, MAX_UINT256). Permanent unrestricted token access — auto-blocked regardless of score.
+70 → FORCE BLOCK
CHECK 05
USD Value
Chainlink PROS/USD feed. Sending >50% of wallet balance surfaces a warning.
+20 if >50%
CHECK 06
Gas Estimation
eth_estimateGas. Catches edge cases the simulation passed but the chain would reject.
+5 if fails

Get Started

Installation

Both skills follow the official Pharos Skill Engine format. Install once, use in any compatible agent.

Prerequisite — Foundry: All cast commands require Foundry's cast CLI.

Install Foundry
# Install Foundry (cast, forge, anvil)
curl -L https://foundry.paradigm.xyz | bash
source ~/.zshenv && foundryup
cast --version  # verify
1
Clone both skills into your skill engine directory
bash
# Security skill
git clone https://github.com/hosein-ul/pharos-tx-guardrail \
  ~/.pharos-skills/pharos-tx-guardrail

# Yield intelligence skill
git clone https://github.com/hosein-ul/pharos-rwa-yield-router \
  ~/.pharos-skills/pharos-rwa-yield-router
2
Register in your Pharos Skill Engine config
Point the engine at each skill's SKILL.md. The YAML frontmatter includes name, description (trigger phrases), and requires.anyBins.
3
Verify connectivity
bash
cast block-number --rpc-url https://rpc.pharos.xyz
# Should return current block (~10M+)
ℹ️ The Pharos Skill Engine reads SKILL.md as the agent entry point, loads assets/networks.json for RPC config, and dispatches to the relevant references/*.md section based on the Capability Index table.
1
Clone into Claude's skills directory
bash
git clone https://github.com/hosein-ul/pharos-tx-guardrail \
  ~/.claude/skills/pharos-tx-guardrail

git clone https://github.com/hosein-ul/pharos-rwa-yield-router \
  ~/.claude/skills/pharos-rwa-yield-router
Skills in ~/.claude/skills/ are auto-discovered by Claude Code. No further config needed.
2
Restart Claude Code / start a new session
Claude will load both skills and auto-trigger them based on the description keywords in each SKILL.md.
3
Test with a prompt
Claude Code chat
# Triggers pharos-tx-guardrail:
"Check if this transaction is safe before I send it..."

# Triggers pharos-rwa-yield-router:
"Where should I put 5000 USDC on Pharos for the best yield?"
ℹ️ Claude Code reads each skill's YAML frontmatter description field to decide when to invoke it — no manual slash command needed.
1
Open the Anvita Flow Skill Hub
Anvita Flow converts Pharos Skills into deployed Agents with one click. Log in at the Anvita Flow platform.
2
Submit the GitHub URLs
Anvita Flow — Skill Hub
# Security Skill URL:
https://github.com/hosein-ul/pharos-tx-guardrail

# Yield Skill URL:
https://github.com/hosein-ul/pharos-rwa-yield-router
3
Deploy as an Agent
Anvita Flow reads SKILL.md, packages both skills, and deploys them as a live Pharos Agent. The agent becomes callable on-chain and accessible via the Pharos ecosystem.
ℹ️ Only Phase 1 hackathon winners can deploy to Pharos via Anvita Flow infra. Skills are submitted via DoraHacks for judging first.
1
Clone both repos
bash
git clone https://github.com/hosein-ul/pharos-tx-guardrail
git clone https://github.com/hosein-ul/pharos-rwa-yield-router
2
Point your agent at SKILL.md
Each skill's SKILL.md is the entry point. Feed it to your agent's system prompt or tool registry. The agent reads the Capability Index table to know which reference file to load.
3
Read network config from assets/networks.json
bash — load RPC from config
RPC=$(jq -r '.networks[] | select(.name=="mainnet") | .rpcUrl' \
  pharos-rwa-yield-router/assets/networks.json)

# Now use $RPC in all cast commands
cast block-number --rpc-url $RPC
ℹ️ Compatible with any agent that can execute shell commands (LangChain tools, OpenAI function calling, AutoGPT, custom orchestrators, etc.). The skill is entirely cast/curl based — no npm, no Python, no dependencies beyond Foundry.
